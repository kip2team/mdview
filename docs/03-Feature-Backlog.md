# mdview · 可扩展功能清单（讨论用）

> 全量功能候选池，按类别组织。每条带价值 / 成本 / 依赖估计，便于一起拍优先级。

价值评估维度：用户感知 + 拉新势能 + 差异化贡献。
成本评估维度：单人开发的总投入（含调试 / 维护）。

图例：

- 价值：★ 一般 / ★★ 高 / ★★★ 关键
- 成本：◐ 小（≤几天） / ◑ 中（1-2 周） / ● 大（数周）
- 状态：✅ 完成 / 🟡 进行中 / 🔲 未开始

## 优先级建议（合并视图）

将上面所有 P0 / P1 / MVP 项整理后给出"先做这些"的清单。下面是我的建议，**最终由你拍板**：

### 第一波（让产品能跑起来 / 第一次发布）
A1, A2, A3, C1, C2, D1, D2, D3, K1

> 大概范围：能解析渲染常见 Markdown，有 3 个主题，桌面端双击即看，沉浸式。这就是"我们做了一个 Markdown 阅读器"的最小可信度。

### 第二波（差异化第一次显现 / 第二次发布）
A4, A6, A7, A8, B1, B3, C3, C4, D7, D8, D9, D10, D12

> 加入了 TOC、callout、色块、键位、暗色、字体调节、融合视图、PDF / .mdv.html 导出。这一波之后产品已经比 Typora / Obsidian 在"阅读体验"维度有差异化感。

### 第三波（mdview.sh 上线 / 第三次发布）
E1, E2, E3, E4, F1, F2, F3, F4, G1, G2, H1, H2, H4, H5, J7, K3

> Web 短链、浏览器扩展接管 GitHub、VS Code 插件、CLI。这一波之后引擎价值兑现，多端联动。

### 第四波（差异化进阶）
B2, B4, B5, B9, B12, C5, D4, D5, D11, F5, F6, G3, H3

> 完善扩展语法、用户自定义、文件夹模式、社交分享。

### 第五波（主题市场 + AI）
C6, C7, C8, E6, E7, I1, I2, I3, I4

> 主题市场、Typora / Obsidian 导入、MCP / AI 增强。

### 第六波（商业化）
C9, C10, C11, E8, E10, E12, J3, J5, K8

> 付费主题、自定义域名、团队空间、企业版。

### 不做 / 慎做
J4（实时协作）、D14（云同步）、E5（在线编辑器深度做）、B14（自定义 HTML 组件）

## 你需要拍板的事项

1. **第一波要不要再砍**？比如 C2 暗色模式可以放到第二波，但用户期望很高
2. **B 类扩展语法**：你想做"标准超集"（保持 .md 在别处也能看）还是"私有方言"（更强但破坏性）？
3. **付费策略时间点**：第六波之前要不要试探 Donate / Patreon 拿一些反馈？
4. **AI 集成**：是放到第五波（差异化）还是更前？AI 现在是流量密码，但单人维护要谨慎
5. **是否做实时协作**（J4）：明确不做的话能省大量精力

## 备忘

- 此清单不是合同，是讨论用的池子。某个功能从"看情况"被升到"1.0"非常正常
- 每完成一波都要复盘是否调整后续优先级
- 单人开发节奏：宁可 6 周做透一波，也不要 2 周铺三波都半成品
